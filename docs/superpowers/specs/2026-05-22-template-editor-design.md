# Design: Template editor (Sub-project 5, slice A of «Шаблоны»)

Дата создания: 2026-05-22
Статус: draft, ждёт ревью пользователя
Связанные документы:
- продуктовая спека `docs/superpowers/specs/2026-05-19-content-system-redesign-design.md` (v7.4) — §2.5 шаблоны, §4.4 «Шаблоны», §4.5 расширенный режим
- планы под-проектов 3–4 (выполнены): generation-core, checking-stages

Это дизайн **первого среза** раздела «Шаблоны» — редактора промтов этапов. Раздел «Шаблоны» большой; он разбит на независимые срезы (см. §6), здесь реализуется только срез A.

## 1. Концепция

Дать редактору возможность **самому менять поведение ИИ** — формулировки промтов, методички (они внутри системного промта), модель и «креативность» (температуру) — без разработчика. Сейчас 6 шаблонов этапов засеиваются кодом и правятся только в коде. Этот срез открывает их для редактирования в разделе «Шаблоны».

## 2. Принятые решения (brainstorming 2026-05-22)

1. **Первый срез — A (редактор промтов этапов).** Остальные срезы «Шаблонов» (ИИ-роли, типы статей, продуктовые блоки, скиллы, песочница) — позже.
2. **Версии шаблонов — Вариант A (правка на месте + сброс).** Сохранение обновляет шаблон и увеличивает счётчик версии; есть «Сбросить к стандартному». Полная история версий с восстановлением — отложена.
3. **Методички** редактируются через системный промт этапа (отдельной сущности «скрытый контекст» не вводим).

## 3. Что редактируется

Для каждого из 6 этапов (Черновик, Продуктовые блоки, Семантика-в-текст, Проверка SEO, Фактчекинг, Финальная вычитка) — существующий `StageTemplate`:
- **Системный промт** (`systemPrompt`) — роль/правила/методичка.
- **Пользовательский промт** (`userPromptTemplate`) — инструкция с переменными `{{…}}`.
- **Параметры модели** (расширенные, скрыты по умолчанию — §4.5): модель (`modelName`), температура (`temperature`), max tokens (`maxTokens`).

Сохранение: обновляет поля, `templateVersion += 1`, `updatedAt = .now`. Прошлые `ArticleVersion` не меняются (они хранят, какой шаблон/модель использовались, — воспроизводимость прошлого сохраняется на уровне снимка).

## 4. Компоненты

- **`StageTemplateDefaults`** (новый, чистая логика) — единый источник стартового содержимого: `content(for: PipelineStage) -> StageTemplateContent`, где `StageTemplateContent` — структура (`systemPrompt`, `userPromptTemplate`, `modelName`, `temperature`, `maxTokens`). Используется и сидером, и кнопкой «Сбросить».
- **`StageTemplateSeeder`** (рефактор) — `makeTemplate(for:)` использует `StageTemplateDefaults.content(for:)` вместо «зашитых» строк. Поведение не меняется (тот же текст).
- **`TemplateVariable`** (новый, статический справочник) — список доступных переменных: имя `{{…}}`, описание, источник (бриф / семантика / база знаний / предыдущая версия / проверки). Только чтение.
- **`TemplatesView`** (новый) — раздел «Шаблоны»: список 6 этапов → редактор (системный/пользовательский промт; «Расширенные»: модель/температура/max tokens; кнопки «Сохранить», «Сбросить к стандартному»); секция/панель «Переменные» (справочник).
- **`RootView`** (правка) — для секции `.templates` показывать `TemplatesView` вместо заглушки.

## 5. Тестирование

- **Unit (TDD):** `StageTemplateDefaults.content(for:)` — есть содержимое для всех 6 этапов, ключевые маркеры на месте (например, у проверяющих — инструкция вернуть JSON; у `semanticsInText` — блок метаданных). Косвенно: `StageTemplateSeederTests` остаются зелёными (сидер использует те же дефолты → по-прежнему 6 шаблонов).
- **Reset-логика:** проверяемо через `StageTemplateDefaults` (сброс = применить дефолтные значения к полям шаблона).
- **Ручной smoke:** открыть «Шаблоны», отредактировать промт этапа, сохранить; запустить этап на теме — генерация использует новый промт; «Сбросить к стандартному» возвращает исходный текст; параметры модели в «Расширенных».

## 6. Декомпозиция раздела «Шаблоны» (для будущих срезов)

- **A. Редактор промтов этапов** — этот под-проект.
- B. ИИ-роли (агенты): контекст и закреплённые этапы (сейчас захардкожены в `PipelineStage`).
- C. Типы статей: создать/редактировать (сейчас фиксированный `ArticleType`).
- D. Продуктовые блоки: библиотека (сейчас фиксированный список в `ProductBlocksSheet`).
- E. Скиллы и песочница: зависят от ручной правки фрагментов (пока не реализована).

## 7. Границы (scope)

В скоупе: правка системного/пользовательского промта и параметров модели у 6 этапов; сохранение на месте с инкрементом версии; «Сбросить к стандартному»; справочник переменных (только чтение); рефактор сидера на единый источник дефолтов; unit-тесты + smoke.

Вне скоупа: полная история версий шаблонов и восстановление; редактор ИИ-ролей; типы статей; продуктовые блоки; скиллы; песочница; вставка переменных кликом/перетаскиванием; импорт/экспорт шаблонов.
