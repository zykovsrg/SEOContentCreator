# Дизайн: каскад ручных промтов → шаблоны приложения

Дата: 2026-07-02
Статус: approved by user (mapping + scope), texts pending user review

## Цель

Перенести ручной каскад из 8 промтов пользователя (редполитика → источники → структура ×2 → черновик ×2 → фактчекинг → свежий взгляд) в дефолтные шаблоны приложения так, чтобы обычный прогон пайплайна воспроизводил каскад без ручного ввода промтов.

## Решения пользователя

1. «Вторые проходы» (шаги 4 и 6 — «сократи») слить в промты этапов «Структура» и «Черновик» + добавить скилл-пресет «Сократить» для ручного повторного прохода по фрагменту.
2. Персона: оставить якорь «копирайтер Т—Ж» + правила Hadassah выжимкой из редполитики.
3. Обновление существующих шаблонов в БД через версионную миграцию — согласовано (ручные правки этих шаблонов затрутся).
4. Таблица запрещённых формулировок: редактируемая внутри приложения (новая модель + UI) — в скоупе.

## Раскладка

| Шаг каскада | Место в приложении |
|---|---|
| 1. Редполитика | `ContextBlock` key `editorialPolicy` — выжимка из Google Doc |
| 2. Источники | `ContextBlock` key `sources` — алгоритм работы с источниками |
| 3+4. Структура | `StageTemplate` stage `structure` |
| 5+6. Черновик | `StageTemplate` stage `draft` |
| 7. Фактчекинг | `StageTemplate` stage `factCheck` (JSON-remarks формат сохраняется) |
| 8. Свежий взгляд | `StageTemplate` stage `finalReview` + `{{запрещённые_формулировки}}` |
| «Сократи» повторно | `SkillPreset` key `shorten`, роль `editor` |
| Персона Т—Ж | mandate роли `author` |

Этапы `productBlocks`, `semanticsInText`, `seoCheck` не меняются.

## Новые тексты

### Роль `author` — mandate

```text
Ты — опытный копирайтер. Ты пишешь медицинские статьи для издания «Т—Ж»: они помогают людям без медицинского образования разобраться в заболеваниях, их лечении и профилактике. Вся информация соответствует принципам доказательной медицины. Пиши достоверно, спокойно и понятно для пациента. Отвечай на русском языке в формате Markdown.
```

### ContextBlock `editorialPolicy` — «Редполитика» (выжимка)

```text
Редполитика Hadassah (кратко):

Задача статьи: помочь читателю решить его проблему, показать преимущества клиники без рекламных преувеличений и соответствовать требованиям поисковиков.

Стиль — инфостиль:
- Один абзац — одна мысль. У каждого предложения есть цель, «воды» нет.
- Простой язык: без терминов, непонятных человеку без медицинского образования (не «патология», а «болезнь»; не «поражение воспалительного характера», а «воспаление»). Новые термины объясняй.
- Избегай страдательного залога, канцелярита, отглагольных существительных, вводных конструкций, скобок, цепочек родительных падежей, сокращений «и т. д.».
- Тон сдержанный, но заботливый: без фамильярности, нравоучений, запугивания и оценочности. К читателю на «вы» (со строчной буквы). Не «ты».

Запрещено:
- «Лучший», «гарантированно», «ведущие специалисты» и любые обещания результата без доказательств.
- Выдумывать факты, статистику, имена врачей, цены и сроки. Только проверяемые факты, а не оценочные суждения.
- Народная медицина и гомеопатия как метод лечения; отрицательно окрашенная лексика («зловонный» → «неприятный»).

Доказательность (E-E-A-T): опирайся на клинические рекомендации, упоминай риски и противопоказания, не назначай лечение.

Типографика: клиника — латиницей «Hadassah»; тире «—», диапазоны «11–12»; кавычки-«ёлочки»; числа до 10 — словами; буква «ё» обязательна.
```

### ContextBlock `sources` — «Источники»

```text
Алгоритм работы с источниками:

1. Начинай с простых источников для пациентов, чтобы сформировать общее понимание темы: UpToDate (раздел для пациентов), MSD Manual (версия для пользователей), NHS, клиника Майо.
2. Затем уточняй нюансы по источникам для врачей: StatPearls, Medscape, международные гайдлайны.
3. Российский контекст обязателен: зарубежная и российская практики отличаются. Проверяй, зарегистрировано ли лекарство в РФ и совпадают ли протоколы. Основной источник — Рубрикатор клинических рекомендаций Минздрава; дополнительно Cochrane.ru и HelixBook (анализы).
4. Для сложных и спорных вопросов — PubMed и сообщества Science-Based Medicine, Quackwatch. Качественная статья опубликована в авторитетном журнале (The Lancet, Nature и т. п.).
5. Лекарства и БАДы: статус — через ГРЛС (лекарство) или Реестр свидетельств госрегистрации (БАД); фармсправочники Drugs.com и RX List; доказательная база — списки ВОЗ, Cochrane, FDA. Выводы формулируй осторожно: «недостаточно доказательств эффективности».
6. Статистика: PubMed, ВОЗ, отчёты ведомств США, Британии, Германии.

Если данных не хватает — обозначай это честно, не заполняй пробелы фантазией.
```

### StageTemplate `structure` (шаги 3+4)

```text
Подготовь структуру текста для посадочной страницы. Только структуру, без текста статьи.

Тема: {{тема}}
Тип статьи: {{тип}}
Целевой объём (знаков): {{объём}}
Направление: {{направление}}
Данные врача: {{врач_данные}}
Преимущества клиники: {{преимущества}}
Приоритетные источники: {{источники_направления}}

Используй H1 и H2 как основные уровни структуры. H3 — только там, где внутри H2 действительно нужен отдельный смысловой подраздел. Под каждым заголовком добавь короткую пометку (1–2 предложения): о чём раздел.

Оставь только самое важное для пациента. Представь, что ты пациент, который выбирает клинику: что тебе было бы важно узнать? В интернете очень много контента, поэтому информацию нужно давать быстро и чётко. Не бойся сокращать структуру.

Не добавляй ключевые запросы. Формат — Markdown.
```

### StageTemplate `draft` (шаги 5+6)

```text
Напиши текст строго по утверждённой структуре.

Тема: {{тема}}
Тип статьи: {{тип}}
Целевой объём (знаков): {{объём}}
Направление: {{направление}}
Данные врача: {{врач_данные}}
Преимущества клиники: {{преимущества}}
Приоритетные источники: {{источники_направления}}

Структура (план статьи):
{{структура}}

Правила текста:
- В тексте должен быть заголовок, который коротко передаёт суть — самое главное, ради чего человек будет читать.
- Подзаголовки — такие, чтобы было понятно, о чём пойдёт речь в блоке. Один абзац — одна мысль. У каждого абзаца и предложения есть цель, «воды» нет.
- Пиши так, чтобы у читателя не оставалось вопросов, для чего он это прочитал.
- Только факты, не оценочные суждения. Если не уверен, что информация соответствует доказательной медицине, — предупреди читателя прямо в тексте.
- Никогда не выдумывай факты, статистику и данные исследований: их должно быть можно проверить поиском. Если сомневаешься — предупреждай. От этого зависит здоровье читателей.
- Не больше трёх списков на весь текст.
- Добавь раздел «## Частые вопросы» на 5–7 вопросов без дублей.
- В конце — раздел «## Источники» на 7–10 позиций без дублей.
- Не пиши заключение или вывод в конце текста или разделов.
- Текст должен быть уникальным, не повторять другие тексты в интернете.

Оставь только самое важное для пациента: представь, что ты пациент, который выбирает клинику. Не бойся сокращать текст.
```

### StageTemplate `factCheck` (шаг 7)

```text
Проведи фактчекинг текста. Сверь медицинские утверждения с принципами доказательной медицины и алгоритмом работы с источниками, а данные клиники — со справочником. Не переписывай текст — верни список замечаний.

Текст:
{{текущий_текст}}

Справочник клиники (База знаний):
{{база_знаний}}

Особое внимание:
- проверяемость фактов, статистики и ссылок на исследования (ничего выдуманного);
- соответствие российскому контексту (регистрация лекарств в РФ, протоколы Минздрава);
- осторожность формулировок о лекарствах и БАДах;
- расхождения с данными клиники (врачи, процедуры, цены).

Верни ТОЛЬКО JSON в формате:
```json
{"remarks":[{"category":"...","quote":"<точный фрагмент из текста>","suggestion":"<доработанный текст>","explanation":"<в чём проблема>"}]}
```
Категории: «Факт», «Статистика», «Источник», «Российский контекст», «Имя врача», «Процедура», «Цена». Поле quote — дословный фрагмент из текста. Если правка не нужна, suggestion оставь пустым.
```

### StageTemplate `finalReview` (шаг 8)

```text
Проверь текст свежим редакторским взглядом. Не переписывай целиком — верни список точечных замечаний.

Текст:
{{текущий_текст}}

Проверь:
- орфографические, пунктуационные, грамматические и стилистические ошибки;
- канцелярит, «воду» и перегруженные предложения;
- шаблонные ИИ-фразы — текст должен звучать естественно;
- лишнее: в тексте должно оставаться только то, что решает проблему читателя. Не бойся предлагать сокращения.

Запрещённые формулировки (замени по образцу):
{{запрещённые_формулировки}}

Верни ТОЛЬКО JSON в формате:
```json
{"remarks":[{"category":"...","quote":"<точный фрагмент из текста>","suggestion":"<доработанный текст>","explanation":"<в чём проблема>"}]}
```
Категории: «Орфография», «Грамматика», «Пунктуация», «Стиль», «Канцелярит», «Вода», «ИИ-штамп», «Запрещённая формулировка». Поле quote — дословный фрагмент из текста.
```

### SkillPreset `shorten` — «Сократить»

```text
Сократи фрагмент: оставь только самое важное для пациента, который выбирает клинику. Убери повторы, необязательные подробности и «воду». Сохрани все факты и смысл, ничего не выдумывай.
```

Роль: `editor`.

## Таблица запрещённых формулировок (новая функция)

### Модель `ForbiddenPhrase` (SwiftData)

- `uuid: UUID`
- `phrase: String` — предложение или формулировка
- `problem: String` — в чём проблема
- `replacement: String` — как можно заменить
- `order: Int`, `createdAt`, `updatedAt`

Регистрация в контейнере приложения и во всех тестовых контейнерах.

### Seeder

`ForbiddenPhraseSeeder.seedIfNeeded` — только если записей нет. Стартовые данные — 5 строк из таблицы пользователя:

| phrase | problem | replacement |
|---|---|---|
| кровянистое отделяемое | плохо звучит | кровянистые выделения |
| сразу после операции пациент находится под наблюдением | недостаточно связи: непонятно, под наблюдением кого | сразу после операции пациент находится под наблюдением врачей |
| в период восстановления обычно рекомендуют не сморкаться с усилием | звучит жаргонно | интенсивно прочищать нос |
| хирург работает через носовые ходы | плохо звучит | хирург проводит операцию через носовые ходы |
| обследования показывают нарушение оттока | недостаточно связи | обследования показывают нарушение оттока слизи |

### UI

Новый раздел «Запрещённые формулировки» в окне «Шаблоны» (`TemplatesView`), по образцу существующих секций: список фраз, редактирование трёх полей, добавление и удаление строк (удаление с подтверждением — по принятому правилу деструктивных действий).

### Подстановка в промт

- Новый плейсхолдер `{{запрещённые_формулировки}}` в `TemplateVariables` (источник: «Шаблоны → Запрещённые формулировки»).
- `StageExecutor` перед вызовом `PromptBuilder` получает записи из контекста и рендерит строки вида `«фраза» — проблема: …; замена: «…»`; передаёт в `PromptBuilder.build` новым параметром.
- Если список пуст — плейсхолдер заменяется на «(список пуст)».

## Словарь клише

В `EditorDictionaryDefaults.clichesText` короткие фразы из таблицы НЕ добавляются: словарь сеется только при пустой БД, а таблица формулировок теперь живёт своей моделью. Дублировать не нужно (см. self-review).

## Доставка в существующую базу (миграция)

`StageTemplateSeeder`: поднять `currentTemplatesDefaultsVersion` до 3. Миграция при `storedVersion < 3`:

1. Для этапов `structure`, `draft`, `factCheck`, `finalReview`: перезаписать `systemPrompt` и `userPromptTemplate` новыми дефолтами.
2. Для контекстных блоков `editorialPolicy`, `sources`: перезаписать `text` (по `key`).
3. Для роли `author`: перезаписать `mandate`.
4. Скилл-пресет `shorten`: добавить, если пресета с таким `defaultKey` нет (через существующий механизм `SkillPresetSeeder`/дефолтов).

Этапы `productBlocks`, `semanticsInText`, `seoCheck`, роли `seo`/`factChecker`/`editor`, блок `seoGuidelines` — не трогать. Пользователь предупреждён, что его ручные правки перечисленных шаблонов будут заменены.

## Тесты

- `ForbiddenPhrase`: seeder идемпотентен; рендер строки для промта; пустой список → «(список пуст)».
- `PromptBuilder`: `{{запрещённые_формулировки}}` подставляется в user-промт.
- Миграция v3: обновляет 4 шаблона, 2 блока и mandate автора; не трогает `seoCheck` и `seoGuidelines`; повторный запуск ничего не меняет.
- Существующие тесты шаблонов/сидеров — зелёные после изменения дефолтов.

## Вне скоупа

- Автозагрузка редполитики из Google Doc (выжимка статична, обновляется руками в «Шаблонах»).
- Отдельные этапы пайплайна для «сокращения» (решено: слияние + пресет).
- Импорт таблицы формулировок из Google Sheets (стартовые 5 строк вшиты, дальше — пополнение в приложении).
