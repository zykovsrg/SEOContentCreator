# Frontend Design: macOS app screens (SwiftUI)

Дата: 2026-05-21
Статус: draft, согласован с пользователем по экранам
Стек: нативное macOS-приложение на SwiftUI (см. `ai/decisions.md`)
Связанный документ: продуктовая спека `docs/superpowers/specs/2026-05-19-content-system-redesign-design.md` (v7.4)

Этот документ фиксирует UI-решения, согласованные через макеты (визуальный brainstorming). Описывает структуру экранов и взаимодействия, не код. Реализация — в SwiftUI; `frontend-design` (веб) не применяется.

---

## 1. Общая структура окна

- **Левый сайдбар разделов** (постоянный): Контент-план, Очередь, Шаблоны, База знаний. Внизу — компактный **виджет очереди** (активная тема, этап, кнопки «Стоп после текущей / Стоп»).
- **Основная область** переключается по разделу (паттерн «список → деталь», master-detail).
- **Сквозной блок этапов** — появляется при открытой теме (полоса сверху рабочей области).

---

## 2. Контент-план (главный экран)

- Таблица тем. Колонки: Тема, Тип, Направление, Статус (последний этап + дата + метка ⏳/⚠/✓), Семантика, Стоимость (₽).
- Фильтры: по этапу, типу, **направлению, врачу**, статусу, «в очереди», «с ошибкой», «опубликовано», поиск.
- Тулбар: «Новая тема», «В очередь», «Повторить упавшие», поиск.
- **Открытие темы — двойной клик** по строке → Рабочее пространство темы; возврат «← Контент-план».

---

## 3. Рабочее пространство темы

- **side-by-side как «полотно темы» (вариант X):** две колонки — постоянный центр, пока работаешь с темой. Слева текущая версия, справа новая с подсвеченными правками.
- **Полоса этапов** сверху: ✓ пройдено, ▶ текущий (подсвечен), дальше — оставшиеся. Переключение этапа меняет текст в колонках с **минималистичной анимацией**.
- Другие разделы (Очередь/Шаблоны/База знаний) открываются отдельно — полотно темы временно скрывается.
- **Шапка темы:** название, тип, направление, врач + кнопки «Запустить этап / В очередь / Пауза».
- **Кнопки принятия:** «Принять всё → след. этап», «Принять частично», «Отклонить».
- **Лента версий** (сайдбар): два вида — «по времени» и «по этапам»; выбор двух версий → diff; «сделать текущей» (откат).
- **Лог темы** (нижняя выдвижная панель): что и какой агент делал.

---

## 4. Приём правок (механика)

- Каждая правка — самостоятельная, со своими **✓ принять / ✗ отклонить**. «Принять частично» собирает **гибридную версию** только из принятого.
- **Пояснения к правкам — панель замечаний справа** (основной способ): карточки по категориям с пояснением «что и почему» + ✓/✗. **Клик по карточке → подсветка и прокрутка к месту в тексте.**
- Текст остаётся **редактируемым** (клик = курсор). Пояснения не в тексте, а в панели — конфликта с курсором нет.
- Модель работает на всех этапах с правками: Стиль, Проверка SEO, Фактчекинг, Финальная вычитка.

---

## 5. База знаний

- Три колонки: разделы / **дерево узлов** / содержимое выбранного узла.
- Дерево: направления → подразделы → факты; кнопки «+ Узел / + Подузел / ✎ / 🗑».
- Детали узла: тип, где используется (N тем), описание, дочерние преимущества, **🔗 приоритетные источники** (у направлений).
- **Поиск** по названиям и содержимому (подсветка).
- **Гибкие фильтры (комбинируются):** тип узла, направления (несколько), использование («в темах» / «сироты»).
- **Сохранённые срезы** (частые комбинации фильтров), переоткрываются.

---

## 6. Бриф темы (форма создания)

Фиксированная форма:
- Название * · Тип статьи * · **Направление** * (из Базы знаний) · Врач (опц., из Базы знаний) · Целевой объём · Параметры (Стиль вкл/выкл) · Заметки (опц.).
- Направление/врач — выбор из узлов Базы знаний (автоприкрепление узлов).
- **Запуск Черновика заблокирован** без названия + типа + направления.

---

## 7. Очередь

- Управление: статус + «Старт / Стоп после текущей / Стоп немедленно / Сброс / Повторить упавшие».
- «Сейчас»: активная тема, этап, прогресс-бар.
- Список в очереди: порядок (перетаскивание), тема, сценарий, статус.
- Лог: фильтр «только ошибки»; ошибки красным, успехи ✓. Упавшая тема не стопорит очередь.

---

## 8. Шаблоны (настройки, power-user)

Категории (понятные названия):
- **Общие правила** — технические для всех агентов (русский, markdown, не выдумывать).
- **Редполитика и источники** — редполитика, источники, SEO-рекомендации.
- **ИИ-роли** — 4 агента: контекст + закреплённые этапы.
- **Промты этапов** — по типам статей, с переменными `{{…}}`, история версий.
- **Продуктовые блоки** — каждый блок отдельный шаблон.
- **Скиллы** — мини-промты для правки фрагмента.
- **Типы статей** — создать/редактировать.

- **Редактор промта** с переменными `{{…}}`; кнопки «Сохранить версию / История / 🧪 Песочница».
- **Реестр переменных** (панель справа): переменные с источником (цвет = источник), клик/перетаскивание вставляет `{{…}}`. Переменная определяется один раз в источнике (бриф / семантика / база знаний / предыдущая версия) — см. спека 2.5.

---

## 9. Сквозные UI-паттерны

- **side-by-side** — основной способ показывать изменения на всех этапах.
- **Сквозная ручная правка** — на любом этапе: выделить фрагмент → регенерировать / применить **скилл** («перепиши в инфостиле»…) / мягкие подсказки (длинные предложения, повторы, канцелярит — алгоритмически, как подчёркивание в Word).
- **Анимация смены этапов** в полотне темы.
- **Статус** — «последний успешный этап + дата + метка», + индикатор семантики.
