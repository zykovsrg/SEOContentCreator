# Design — Читаемая «Формула задачи» независимо от формулировок

Дата: 2026-07-24
Статус: одобрено пользователем

## Проблема

`ReaderIntent.taskFormula` собирает одну плавную фразу
(«Помочь `<audience>` `<goal>`, чтобы `<success>`, учитывая `<barriers>`,
в формате: `<format>`.») из свободных текстовых полей карты задачи читателя.
Поля («Обновить с ИИ» или ручной ввод) обычно содержат ЦЕЛЫЕ предложения со своей
пунктуацией и заглавными буквами, поэтому фраза ломается:
«Помочь **Вероятно, читатель** столкнулся… Кибаку**., чтобы** Читатель получает…».

Формула показывается только в `ReaderIntentSheet` («Формула задачи»). В промт ИИ-этапов
идёт ОТДЕЛЬНЫЙ `ReaderIntentPromptRenderer.render` (список с подписями), поэтому правка
формулы НЕ влияет на генерацию — это чисто читаемость.

## Решение

Заменить плавную фразу на набор подписанных частей-предложений (детерминированно, без ИИ).
Каждое непустое поле → своя часть `«<Ярлык>: <значение>»`; части соединяются пробелом.

Ярлыки и порядок:
- **Кому** — audienceContext (пропускается, если пусто)
- **Задача** — hiddenGoal (ВСЕГДА; если пусто — «решить практическую задачу»)
- **Польза** — successCriterion (если непусто)
- **Барьеры** — barriers (если непусто)
- **Формат** — solutionFormat (если непусто)

Нормализация конца каждой части (чтобы не было `..` и висящих запятых):
- значение обрезается по пробелам (существующий `clean`);
- если кончается на `.` `!` `?` `…` — оставить как есть;
- если кончается на `,` `;` `:` — заменить последний символ на `.`;
- иначе — добавить `.`.

Значения НЕ меняем по регистру: «Кому: Вероятно, читатель…» читается нормально как
подписанная часть.

### Пример

Было:
> Помочь Вероятно, читатель столкнулся… хирургу. Понять… Кибаку., чтобы Читатель
> получает…, учитывая Возможны опасения…, в формате: Страница услуги….

Станет:
> Кому: Вероятно, читатель столкнулся… хирургу. Задача: Понять… Кибаку. Польза: Читатель
> получает… на услугу. Барьеры: Возможны опасения… аудитории. Формат: Страница услуги…
> к профильному врачу.

(В `ReaderIntentSheet` ярлыки можно оставить внутри строки как есть — жирное начертание
не обязательно; строка `taskFormula` остаётся plain text.)

## Область изменений

- `ReaderIntent.taskFormula(...)` (static) — единственная реализация; `ReaderIntent.taskFormula`
  (computed) и `ReaderIntentDraft.taskFormula` уже делегируют сюда. Одно место.
- НЕ трогаем `ReaderIntentPromptRenderer` (влияет на промт) и `ReaderIntentSheet` (показ строки
  не меняется структурно).

## Тесты (ReaderIntentTaskFormulaTests, новый файл)

- поле-предложение с точкой на конце → нет `..`;
- поле с запятой/двоеточием на конце → заменяется на `.`;
- фрагмент без пунктуации → добавляется `.`;
- пустые Польза/Барьеры/Формат → части пропущены;
- пустой hiddenGoal → «Задача: решить практическую задачу.»;
- всё пусто → только «Задача: решить практическую задачу.».

## Риски

Минимальные: display-only, детерминированно, одно место, покрыто тестами. `taskFormula`
дублируется как computed/бридж — держим одну static-реализацию.
