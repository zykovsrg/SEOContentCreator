# Design: Генерация изображений (обложка, иллюстрации, пресеты стиля)

Дата: 2026-05-24
Под-проект: 8 (изображения)
Статус: согласован, готов к плану

## 1. Цель

Дать редактору генерировать изображения для статьи прямо в приложении:

- **Обложку** статьи;
- **Иллюстрации** к конкретным фрагментам текста;
- в едином **фирменном стиле** (палитра, ограничения, аудитория), переиспользуемом между темами;
- с возможностью **дорабатывать** сгенерированную картинку инструкцией.

Сейчас изображений в продукте нет (в основной спеке `2026-05-19` они не описаны) — это новая фича. Логика следует общей философии приложения: ИИ-инфраструктура переиспользуется, библиотеки (пресеты/шаблоны) редактируемы, история не затирается.

Картинки хранятся как **галерея у темы** (отдельно от ленты текстовых версий). Текст статьи остаётся чистым. Каждая иллюстрация запоминает **место** (цитату-якорь) для будущей вставки в Google Docs.

## 2. Объём

В объёме:
- Генерация обложки и иллюстраций через OpenAI images API (имя модели настраиваемое: `gpt-image-1` / `gpt-image-2` / иное).
- Сущность «Сгенерированное изображение» — галерея у темы.
- Сущность «Пресет стиля» — редактируемая библиотека (стиль-текст + опц. референс-картинка + параметры). Засев одним дефолтным брендовым пресетом.
- Сущность «Шаблон промта изображения» — редактируемые шаблоны сюжета для обложки и иллюстрации, с переменными. Засев дефолтами.
- Новая переменная `{{выделенный_фрагмент}}`.
- `ImageClient` (запрос к images API) + `ImageGenerator` (сборка промта, вызов, сохранение, лог).
- **Доработка** картинки инструкцией (новая картинка-производная, исходник не затирается).
- UI: панель «Изображения» в рабочем пространстве (галерея + генерация + доработка + экспорт PNG); категория «Изображения» в разделе «Шаблоны»; поле image-модели в Настройках.

Вне объёма (отдельные под-проекты / будущее):
- **Вставка картинок в Google Docs** — будущий этап «Публикация». Сейчас: храним якорь-цитату + экспорт PNG вручную.
- Захват выделения из редактора — фрагмент **вставляется вручную** в поле окна.
- **Маска области / inpainting** — доработка только инструкцией (промтом) по всей картинке.
- Апскейл, вариации, серийная авто-генерация иллюстраций по всей статье.
- Встраивание картинок в markdown текста версий (галерея — отдельная сущность).
- Видео/анимация.

## 3. Модель данных

### 3.1 GeneratedImage (новый @Model)
- `uuid: UUID`
- `roleRaw: String` — `ImageRole`: `cover` / `illustration`
- `data: Data` с `@Attribute(.externalStorage)` — байты PNG (внешними файлами, не раздувают БД)
- `promptUsed: String` — финальный промт, отправленный в API (воспроизводимость)
- `presetID: UUID?` + `presetNameSnapshot: String?` — применённый пресет (снимок имени)
- `anchorQuote: String?` — для иллюстрации: фрагмент-якорь (у обложки `nil`)
- `sourceImageID: UUID?` — для доработки: из какой картинки сделана (иначе `nil`)
- `modelName: String?` — имя image-модели на момент генерации
- `createdAt: Date`, `isArchived: Bool`
- `@Relationship var topic: Topic?`

Свойство `role: ImageRole` — get/set поверх `roleRaw` (по образцу `ArticleVersion.source`).

### 3.2 ImageStylePreset (новый @Model)
- `uuid: UUID`, `name: String`
- `styleText: String` — переиспользуемый блок стиля: палитра, ограничения, аудитория
- `referenceImageData: Data?` с `@Attribute(.externalStorage)` — опц. референс-картинка
- `size: String` — напр. `1024x1024`, `1536x1024`, `1024x1536`
- `quality: String` — напр. `high` / `medium` / `low` (зависит от модели; сверить с докой)
- `createdAt: Date`, `updatedAt: Date`

Засев одним дефолтным пресетом (`ImageStylePresetDefaults`):
```
Палитра (используй эти цвета): #F4F9FF, #E8F1FF, #D9E8FD, #007AC0.
На изображении не должно быть текста.
Иллюстрация для пациента: объясняет медицинскую информацию людям без специальных знаний в медицине.
Чистый, аккуратный, дружелюбный медицинский стиль.
```

### 3.3 ImagePromptTemplate (новый @Model)
- `uuid: UUID`
- `kindRaw: String` — `ImagePromptKind`: `cover` / `illustration`
- `userPromptTemplate: String` — шаблон сюжета с переменными
- `createdAt: Date`, `updatedAt: Date`

Дефолты в `ImagePromptDefaults`, засев идемпотентный (по образцу `StageTemplateSeeder`):
- **Обложка** (`{{тема}}`, опц. `{{структура}}`):
  ```
  Сгенерируй обложку для медицинской статьи.
  Тема статьи: {{тема}}
  Обложка должна отражать тему и быть уместной для медицинского сайта клиники.
  ```
- **Иллюстрация** (`{{тема}}` + `{{выделенный_фрагмент}}`):
  ```
  Сгенерируй иллюстрацию для статьи: {{тема}}
  Хочу проиллюстрировать эту часть текста:
  {{выделенный_фрагмент}}
  ```

Стиль (палитра, «без текста», аудитория) в шаблон НЕ зашит — он приходит из пресета (см. §4). Финальный промт = шаблон сюжета + `\n\n` + `styleText`.

### 3.4 Изменения у Topic
- `@Relationship(deleteRule: .cascade, inverse: \GeneratedImage.topic) var images: [GeneratedImage]`
- `coverImageID: UUID?` — выбранная «текущая» обложка (по образцу `currentVersionID`; история обложек сохраняется)

### 3.5 Новая переменная
`TemplateVariables` дополняется `{{выделенный_фрагмент}}` (описание: «Фрагмент текста для иллюстрации», источник: «Окно генерации иллюстрации»). В `Topic` не хранится — передаётся при генерации.

### 3.6 Миграция
Все изменения — новые `@Model` и поля со значениями по умолчанию → лёгкая миграция SwiftData. Новые типы добавляются в схему `ModelContainer`. У старых тем пустая галерея, `coverImageID = nil`.

## 4. Слой логики

### 4.1 Построение промта (чистая функция, тестируемая)
Вход: шаблон сюжета + тема + опц. фрагмент + пресет. Выход: строка финального промта + параметры запроса (size/quality/model, опц. референс/исходник).

Финальный промт:
```
[заполненный шаблон сюжета]

[styleText пресета]
```
Подстановка переменных — та же `{{...}}`-механика. `{{выделенный_фрагмент}}` подаётся отдельным значением: `PromptBuilder.build` получает опц. параметр `extras: [String: String] = [:]` (текстовый пайплайн не затрагивается), либо отдельный `ImagePromptBuilder` (решить в плане — предпочтительно расширить `PromptBuilder`, чтобы не дублировать подстановку).

### 4.2 ImageClient (по образцу OpenAIClient, без стриминга)
- Один запрос/ответ.
- Выбор endpoint:
  - есть референс пресета **или** исходник для доработки → `/v1/images/edits` (картинки на вход + промт);
  - иначе → `/v1/images/generations`.
- Тело: `model`, `prompt`, `size`, `quality`, `response_format=b64_json`, `n=1`; для edits — изображение(я) как multipart.
- Ответ: декодируем base64 → `Data`.
- Ошибки: переиспользуем `OpenAIClient.OpenAIError` (401/429/HTTP/разбор) с читаемыми текстами.
- Инъекция `URLSession` для тестов (canned ответы через `URLProtocol`, как в `OpenAIClientTests`).

### 4.3 ImageGenerator (по образцу StageExecutor, @MainActor @Observable)
Наблюдаемое: `isRunning`, `lastErrorMessage`, `lastResultImageID`.

**Режим создания** (cover/illustration):
1. Строит финальный промт (§4.1).
2. Вызывает `ImageClient` (generations или edits при референсе).
3. Успех → `GeneratedImage(role, data, promptUsed, presetID/snapshot, anchorQuote для иллюстрации, modelName)`; добавляет в `topic.images`; если роль `cover` и `topic.coverImageID == nil` — ставит `coverImageID`. Пишет `GenerationJob` (success).
4. Ошибка/нет ключа → `GenerationJob` (error), `lastErrorMessage`, картинка не создаётся.

**Режим доработки**:
- Вход: исходный `GeneratedImage` + инструкция-промт.
- Промт = инструкция + `\n\n` + `styleText` пресета; endpoint `edits` с байтами исходника.
- Успех → новый `GeneratedImage(sourceImageID = исходник.uuid)`, роль и `anchorQuote` наследуются от исходника. Исходник не меняется.
- Доработку можно дорабатывать дальше (цепочкой): результат — тоже `GeneratedImage`, который снова можно открыть «Доработать».

Зависимости инъектируются (provider изображений + `keyProvider`) — для тестов canned-провайдер возвращает `Data` (по образцу `cannedStream`).

### 4.4 Лог запусков
Каждая генерация/доработка пишет `GenerationJob` (для лога), но **не** создаёт `ArticleVersion`. Имя «агента» в job — напр. «Генератор изображений». Историю job не трогаем (снимки).

## 5. Интерфейс

### 5.1 Панель «Изображения» в рабочем пространстве темы
- Галерея миниатюр: обложки и иллюстрации (пометка роли; у иллюстраций — якорная цитата; у доработок — связь с исходником).
- Кнопки: «Сгенерировать обложку», «Сгенерировать иллюстрацию».
- Действия над картинкой: открыть крупно, «Сохранить файл» (экспорт PNG), «Доработать», «Сделать обложкой» (для обложек), «В архив».

### 5.2 Окно генерации (ImageGenerationSheet)
Общее для обложки/иллюстрации/доработки, режим задаётся при открытии.
- Поле промта-сюжета, предзаполненное из шаблона (для иллюстрации — отдельное поле фрагмента, куда **вставляешь** нужный кусок текста; оно подставляется в `{{выделенный_фрагмент}}`).
- Выбор пресета стиля (показывает палитру/референс).
- Параметры размера/качества (из пресета, разово переопределяемы).
- В режиме доработки: показ исходной картинки + поле инструкции.
- «Сгенерировать» → спиннер «Генерация…» → превью → «Сохранить в галерею».
- При ошибке — красная плашка с читаемым текстом (как в рабочем пространстве этапов).

### 5.3 Категория «Изображения» в разделе «Шаблоны»
Рядом с «Промты этапов», «ИИ-роли», «Редполитика и источники»:
- Редактор шаблонов промта: обложка / иллюстрация + «Сбросить к стандартному».
- Управление пресетами стиля: CRUD (имя, `styleText`, загрузка/очистка референс-картинки, размер/качество) + «Сбросить к стандартному» для дефолтного пресета.

### 5.4 Настройки
Поле имени image-модели (дефолт `gpt-image-1`), рядом с выбором chat-модели. Хранение — `@AppStorage` (по образцу `openAIModel`).

## 6. Потоки данных

**Обложка:** кнопка → окно (обложка) → промт из шаблона + дефолтный пресет → правка → «Сгенерировать» → `ImageGenerator` (создание) → превью → «Сохранить» → `GeneratedImage(cover)`, при первой обложке ставится `coverImageID`; `GenerationJob`.

**Иллюстрация:** кнопка → окно (иллюстрация) → вставляешь фрагмент → `{{выделенный_фрагмент}}` → пресет/правка → «Сгенерировать» → `GeneratedImage(illustration, anchorQuote = фрагмент)`; `GenerationJob`.

**Доработка:** «Доработать» на картинке → окно (доработка, показывает исходник) → инструкция → «Сгенерировать» → `edits` с байтами исходника → новый `GeneratedImage(sourceImageID)`; роль/якорь наследуются; `GenerationJob`.

## 7. Обработка ошибок
- Нет API-ключа → «Укажите API-ключ в Настройках»; job=error; картинки нет.
- 401/429/HTTP/разбор → читаемый текст `OpenAIError` в окне; job=error; ничего не сохраняется.
- Закрытие окна во время генерации → запрос отменяется (по образцу `StageExecutor` `onTermination`).
- Несовпадение параметров `gpt-image-2` → всплывёт как HTTP-ошибка с понятным текстом; чинится сменой модели/параметров в Настройках/пресете (без правки кода).
- Якорь: на этапе генерации только сохраняем цитату; поиск места (найдено/не найдено) — забота будущего этапа «Публикация».

## 8. Тестирование (TDD)
- `ImagePromptDefaults`: контент для обоих видов; иллюстрация содержит `{{выделенный_фрагмент}}`, обложка — `{{тема}}`; стиль НЕ зашит в шаблон.
- Построение промта: подстановка `{{тема}}` и `{{выделенный_фрагмент}}`; сборка «шаблон + `styleText`» в правильном порядке; пустой фрагмент не ломает.
- Засев: один дефолтный пресет + один шаблон на вид; повторный засев не дублирует; миграция добавляет без дублей.
- `ImageGenerator` (создание): успех создаёт `GeneratedImage` с верными role/data/promptUsed/anchorQuote и пишет job; первая обложка ставит `coverImageID`; нет ключа → job=error, картинки нет; API-ошибка → job=error.
- `ImageGenerator` (доработка): создаёт новый `GeneratedImage` с `sourceImageID`, наследует роль/якорь, пишет job; исходник не меняется.
- `ImageClient`: generations без референса; edits с референсом и с исходником; тело включает size/quality/model/b64_json; парсинг b64 → `Data`; маппинг 401/429/HTTP → `OpenAIError` (зеркало `OpenAIClientTests`).
- `TemplateVariables`: `{{выделенный_фрагмент}}` с описанием/источником.
- `Topic`: связь `images`; `coverImageID` резолвит обложку.
- `ImageRole` / `ImagePromptKind`: round-trip rawValue; русские заголовки.

HTTP мокается через `URLProtocol`/canned session.

## 9. Риски и заметки
- **Знания до августа 2025:** точные параметры `gpt-image-2` (значения quality, поведение endpoint, формат multipart для edits) надо сверить с актуальной докой OpenAI при реализации. Имя модели и параметры вынесены в настройки/пресет — адаптация это правка конфигурации, не архитектуры.
- SwiftData: 3 новых `@Model` + поля у `Topic` — лёгкое расширение схемы; существующие данные сохраняются. Новые типы добавить в схему `ModelContainer` (и в in-memory контейнеры тестов).
- `@Attribute(.externalStorage)` для `Data` — байты во внешних файлах; следить за размером при множестве картинок (архивация вместо удаления).
- Запуск юнит-тестов из CLI в этой среде зависает; проверка компиляции — `xcodebuild build-for-testing`, прогон тестов — Cmd+U в Xcode.
- Стоимость: генерация изображений платная; `n=1`, без авто-серий — расход под контролем пользователя.

## 10. Критерий готовности
- Можно сгенерировать обложку и иллюстрацию (с вставкой фрагмента), результат попадает в галерею темы.
- Пресет стиля применяется; правка пресета влияет на новые генерации; шаблоны промта редактируются и «Сбрасываются к стандартному».
- Доработку картинки можно сделать инструкцией; создаётся новая картинка, исходник цел.
- «Сделать обложкой», экспорт PNG, архивация работают.
- Иллюстрация хранит якорь-цитату (для будущей вставки в Docs).
- Компиляция зелёная; юнит-тесты (Cmd+U) зелёные; smoke пользователем пройден.
