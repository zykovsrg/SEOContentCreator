# Soft Editing Hints — Design

Date: 2026-06-24
Source task: FT-20260623-003 (раздел 7 спеки), **только пункт 3**
Status: approved by user 2026-06-24

## Context

Карточка FT-20260623-003 описывает три инструмента ручной правки. Пользователь
решил реализовать **только пункт 3 — мягкие алгоритмические подсказки** (без ИИ).
Пункты 1 (пресеты скиллов) и 2 (регенерация фрагмента) остаются в
`ai/future-tasks.md` со статусом `promoted`/`idea` и в этот scope не входят.

Технический контекст из кода:

- Рабочее пространство темы (`Views/TopicWorkspaceView.swift`) показывает текст
  только для чтения: `HighlightedText` (подсветка одного вхождения) и
  `SideBySideView` (read-only diff). Полноценного редактируемого `TextEditor`
  с программным выделением фрагмента нет — поэтому пункты 1–2 отложены.
- Правки текста идут через систему `Remark` (`quote → suggestion`,
  `RemarkApplier`). Мягкие подсказки её **не** используют и не дублируют
  «Финальную вычитку» (этап `finalReview`).
- Дополнительные окна (Семантика, Структура, Изображения) открываются как
  `.sheet` из тулбара — этот же паттерн используем для подсказок.
- Редактируемые сущности (`StageTemplate`, `AIRole`, `ContextBlock`,
  `ImageStylePreset`) живут в `Views/TemplatesView.swift` и засеваются
  сидерами (`StageTemplateSeeder` и т.п.).

## Goal

Мягкие алгоритмические подсказки без ИИ на текущем тексте темы:

1. Длинные предложения (по настраиваемому порогу слов).
2. Повторы однокоренных слов рядом (в окне из N слов).
3. Штампы по редактируемому словарю фраз.

Подсказки видны при просмотре, **не блокируют сохранение**, грубые и мгновенные
(синхронный анализ, без сети). Чисто информативно: подсветка + список, без
авто-исправлений и без создания версий.

## Non-goals (YAGNI / out of scope)

- Никакого ИИ, генерации, изменения версий или системы `Remark`.
- Никаких авто-исправлений — только подсветка и пояснения.
- Пункты 1 (пресеты скиллов) и 2 (регенерация фрагмента) — не в этом scope.
- Полноценная морфология русского языка (стеммер) — не делаем; используем
  прагматичный префиксный детектор (см. ниже).
- Импорт словаря из файла — не делаем (только ручное редактирование текста).

## Architecture

### Размещение
Кнопка **«Подсказки»** в тулбаре `TopicWorkspaceView` → `@State showHints` →
`.sheet { SoftHintsSheet(topic:) }`. Окно: слева текст текущей версии с цветной
подсветкой, справа панель со списком подсказок (тап → подсветка места).
Существующий экран (review / SideBySide) не трогаем — отдельное состояние.

### Анализатор (ядро, без UI)
`Logic/SoftHints.swift` — чистая функция:

```
enum SoftHintKind { case longSentence, repeatedRoot, cliche }

struct SoftHint: Identifiable {
    let id = UUID()
    let kind: SoftHintKind
    let range: Range<String.Index>   // диапазон в анализируемом тексте
    let message: String              // короткое пояснение на русском
}

enum SoftHints {
    static func analyze(text: String, settings: SoftHintsSettings) -> [SoftHint]
}

struct SoftHintsSettings {
    var longSentenceWordLimit: Int   // по умолчанию 30
    var repeatWindowWords: Int       // по умолчанию 30
    var cliches: [String]            // фразы-штампы
}
```

Чистая, детерминированная, без зависимостей от SwiftData/SwiftUI — полностью
покрывается юнит-тестами. `SoftHintsSettings` собирается из `EditorDictionary`
вызывающей стороной (sheet), чтобы анализатор не знал про модель данных.

### Алгоритмы

**Длинные предложения.** Текст режется на предложения по `.`, `!`, `?`, `…`
и переводам строк. Слово = последовательность букв/цифр. Если число слов в
предложении > `longSentenceWordLimit` — хинт на весь диапазон предложения.

**Повторы однокоренных рядом (прагматичный, без стеммера).** Слова длиннее
4 букв нормализуются (нижний регистр, ё→е). «Корень» ≈ первые 5 букв
нормализованной формы. Идём окном `repeatWindowWords` слов: если две формы
с одинаковым «корнем» попадают в окно — помечаем вторую (диапазон второго
слова), в сообщении указываем первое. Грубо, возможны ложные срабатывания —
это приемлемо («грубые и мгновенные»). Стоп-слова (короткие/служебные) не
учитываются за счёт порога длины. Чтобы не плодить шум: одинаковые точные
словоформы тоже считаются повтором (это частный случай общего корня).

**Штампы.** Каждая непустая фраза словаря ищется в тексте без учёта регистра
(ё→е), по границам слов. Все вхождения → хинты с диапазоном вхождения.

### Хранение словаря и настроек
**Новая** SwiftData-модель `Models/EditorDictionary.swift` — одна засеянная
запись приложения:

```
@Model final class EditorDictionary {
    var uuid: UUID
    var clichesText: String          // фразы, по одной на строку
    var longSentenceWordLimit: Int   // 30
    var repeatWindowWords: Int       // 30
    var version: Int
    var updatedAt: Date
}
```

`clichesText` парсится в `[String]` (строки, trimmed, без пустых) при сборке
`SoftHintsSettings`.

`Logic/EditorDictionarySeeder.swift` — `seedIfNeeded(in:)`, по образцу
`StageTemplateSeeder`. Дефолтный список штампов и пороги — в том же файле или
в `EditorDictionaryDefaults`.

> **Риск (storage / data model).** Это изменение схемы SwiftData. Но модель
> **новая и отдельная**: добавляется пустая таблица, существующие сущности
> (`Topic`, `StageTemplate`, `ArticleVersion` и т.д.) не меняются. SwiftData
> обрабатывает добавление нового типа без ручной миграции. Риск низкий.
> Модель нужно зарегистрировать в `ModelContainer` (там же, где остальные
> `@Model`), и засеять при старте (рядом с существующими сидерами).

### Редактирование в «Шаблонах»
В `Views/TemplatesView.swift` добавить секцию «Словарь правок» с одним пунктом.
Редактор (по образцу `ContextBlockEditorView`):

- `TextEditor` для штампов (по одной фразе на строку);
- два `Stepper` — порог длины предложения и окно повторов;
- «Сохранить» (инкремент `version`, `updatedAt`);
- «Сбросить к стандартному» (значения из `EditorDictionaryDefaults`).

`TemplateSelection` получает новый case `.editorDictionary(UUID)`; `detail`
и `ensureSelection` дополняются соответствующей веткой. Засев гарантирует
наличие записи перед открытием раздела.

### Мульти-подсветка
Текущий `HighlightedText` подсвечивает одно вхождение одной строки. Для подсказок
нужна подсветка нескольких диапазонов разными цветами по типу хинта. Реализуем
**отдельный** `MultiHighlightedText` (или расширим `HighlightedText` доп.
инициализатором), чтобы не ломать текущее использование в review-режиме.

- Вход: текст + массив `(Range<String.Index>, Color)`.
- Строит `AttributedString`, ставит `backgroundColor` по каждому диапазону.
- Цвета по типу: длинное предложение — например жёлтый; повтор — оранжевый;
  штамп — розовый/красный (точные оттенки уточняются при реализации, без
  привязки к hardcoded значениям спеки).
- Выбранный в панели хинт подсвечивается ярче (как `highlightedQuote` сейчас).

### Панель подсказок
По образцу `RemarksPanelView`: заголовок «Подсказки: N», список карточек
(тип + сообщение), тап → выделить соответствующий диапазон в тексте.
Если подсказок нет — `ContentUnavailableView` («Подсказок нет»).

## Data flow

1. Пользователь жмёт «Подсказки» → открывается `SoftHintsSheet(topic:)`.
2. Sheet берёт `topic.currentVersion?.text ?? ""`, читает `EditorDictionary`
   (fetch + seed-fallback, как `fetchTemplate`), собирает `SoftHintsSettings`.
3. Вызывает `SoftHints.analyze(text:settings:)` → `[SoftHint]`.
4. Слева `MultiHighlightedText` рисует подсветку; справа панель со списком.
5. Тап по карточке → `selectedHintID` → диапазон подсвечивается ярче.
6. Закрытие окна ничего не сохраняет.

## Error handling / edge cases

- Пустой или очень короткий текст → пустой массив хинтов, экран «Подсказок нет».
- Пустой словарь штампов → блок штампов не даёт хинтов (длинные/повторы работают).
- Текст без знаков конца предложения → считается одним предложением.
- Перекрывающиеся диапазоны (штамп внутри длинного предложения) → допускаются;
  при построении `AttributedString` цвета накладываются по порядку применения
  (последний выигрывает по фону); порядок применения фиксируем (предложения →
  повторы → штампы), чтобы результат был детерминированным.
- Диапазоны считаются в индексах **того же** текста, что отдаётся в
  `MultiHighlightedText` (никаких рассинхронов индексов).

## Testing

Юнит-тесты на `SoftHints.analyze` (чистая функция, тестируется без UI):

- длинное предложение засекается, короткое — нет (граница порога);
- повтор однокоренных в окне засекается; те же слова далеко друг от друга — нет;
- точный повтор словоформы засекается;
- штамп из словаря находится без учёта регистра и ё/е; отсутствующий — нет;
- пустой текст / пустой словарь → корректное поведение без падений;
- диапазоны валидны (внутри границ текста).

Сборка: `xcodebuild build-for-testing` (зелёная). Прогон тестов — в Xcode
(Cmd+U), т.к. CLI `xcodebuild test` зависает (см. память окружения).

UI-проверка вручную: открыть тему с текстом, нажать «Подсказки», убедиться, что
подсветка и список появляются, тап работает, закрытие ничего не меняет;
отредактировать словарь в «Шаблонах» и увидеть эффект.

## Done criteria

- «Подсказки» открывают окно с подсветкой длинных предложений, повторов
  однокоренных и штампов на тексте текущей версии.
- Словарь штампов и пороги редактируются в «Шаблонах», сброс к стандартному
  работает.
- Подсказки ничего не сохраняют и не блокируют работу.
- Существующие данные не теряются (новая модель — без миграции существующих).
- `xcodebuild build-for-testing` зелёный; тесты `SoftHints` зелёные в Xcode.
