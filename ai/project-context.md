# Project Context

Проект: внутренний macOS-инструмент для производства медицинских SEO-статей клиники.

## Что это за проект

Персональная (single-user) писательская студия для одного редактора. Ведёт каждую тему статьи по пайплайну от брифа до публикации: сбор семантики, генерация и доработка текста несколькими ИИ-агентами, проверки (SEO, фактчекинг, вычитка), финальная публикация в Google Docs. Цель — производить медицинские SEO-статьи быстро и с контролем качества и фактов.

Статус: дизайн завершён, идёт поэтапная реализация (7 под-проектов). Реализован под-проект 1 «Фундамент» (SwiftUI). Полная спека — `docs/superpowers/specs/2026-05-19-content-system-redesign-design.md`.

## Стек

Нативное macOS-приложение на **SwiftUI** (решение от 2026-05-21, см. `ai/decisions.md`). Веб-обёртка отклонена (нужен только macOS). Потребуется Xcode. Skill `frontend-design` (веб) напрямую не применяется — UI проектируется макетами и реализуется в SwiftUI.

## Как запустить локально

Открыть `SEOContentCreator/SEOContentCreator.xcodeproj` в Xcode → Cmd+R.

## Как собрать

`cd SEOContentCreator && xcodebuild -scheme SEOContentCreator -destination 'platform=macOS' build`

## Как запустить тесты

`cd SEOContentCreator && xcodebuild test -scheme SEOContentCreator -destination 'platform=macOS' -only-testing:SEOContentCreatorTests`

## Главные папки и файлы

- `SEOContentCreator/SEOContentCreator.xcodeproj` — Xcode-проект (схема SEOContentCreator).

## Главные экраны или модули

(спека, раздел 4)
- Контент-план — список всех тем, фильтры, точка входа.
- Рабочее пространство темы — side-by-side «полотно темы» (текущая vs новая версия).
- Очередь и automation — фоновая последовательная обработка.
- Шаблоны — промты этапов, ИИ-роли, продуктовые блоки, пресеты.

## Инварианты проекта

Правила, которые нельзя ломать.

- Single-user (один редактор, без ролей и блокировок).
- Google Docs — только финальная публикация, без двусторонней синхронизации.
- Версии текста не удаляются, только архивируются.
- Публикация — только по явному действию (нет авто-публикации).
- ИИ не переписывает текст молча: проверки показывают предложения, человек принимает.
- Проверяющие ИИ-роли отделены от автора (разные мандаты, вход, знания).
- Пайплайн нежёсткий: любой этап можно запустить в любой момент.
- Ручная правка — сквозная, доступна на любом этапе.
- Черновик нельзя запустить без названия, типа статьи и направления.

## Хрупкие зоны

- Кода нет. На этапе реализации обратить внимание на: согласованность единой ленты версий, фактчек по Базе знаний, корректную остановку очереди при ошибках, отсутствие обратной синхронизации с Docs.
