### 2026-06-26 — Продуктовые блоки из Базы знаний (FT-20260623-002)

- Change: Хардкод из 4 продуктовых блоков в `ProductBlocksSheet` заменён редактируемыми шаблонами. Новая аддитивная SwiftData-модель `Models/ProductBlock.swift` (uuid/name/prompt/order) со стартовым набором (`Logic/ProductBlockDefaults.swift`: CTA «Записаться» / Почему мы / Блок врача / Преимущества клиники) и сидером (`Logic/ProductBlockSeeder.swift`, вызывается в `RootView`). `Logic/PromptBuilder.swift` переработан: словарь подстановок выделен в локальный `substitute(_:)` и применяется и к шаблону стадии, и к промту каждого выбранного блока — KB-переменные (`{{врач_данные}}`, `{{преимущества}}` и т.д.) раскрываются внутри промтов блоков. Собранные промты вставляются через плейсхолдер `{{продуктовые_блоки}}`; при его отсутствии (старые сохранённые шаблоны) — fallback-дописывание в конец. Дефолт стадии `productBlocks` в `StageTemplateDefaults` получил `{{продуктовые_блоки}}`. `ProductBlocksSheet` строит список из `@Query ProductBlock` и отдаёт промты (не имена) через неизменённую сигнатуру `onGenerate: ([String]) -> Void` (call site в `TopicWorkspaceView` не тронут). Секция «Продуктовые блоки» в `TemplatesView` (CRUD + сброс к стандартному, по образцу «Скиллы»). Тесты: +4 в `PromptBuilderTests` (вставка через плейсхолдер, KB-переменные внутри блока, fallback-append, очистка плейсхолдера при пустом выборе), +2 в `ProductBlockSeederTests` (сидирование при пустой таблице, идемпотентность). 8 коммитов 6cb7bfc..6be6cf5 на ветке `feature/product-blocks-kb`, влита в `main`.
- Impact: Продуктовые блоки теперь редактируются пользователем в «Шаблонах», а их данные (врач, преимущества) берутся из прикреплённых узлов Базы знаний темы, а не выдумываются ИИ. Новая схема SwiftData — лёгкая миграция (новая сущность, без явного MigrationPlan; существующие данные не затрагиваются). Любой `ModelContainer` (включая тестовые) должен регистрировать `ProductBlock`. Старые сохранённые шаблоны стадии без `{{продуктовые_блоки}}` продолжают работать через fallback — ломать fallback нельзя.
- Manual checks: `xcodebuild build-for-testing` — TEST BUILD SUCCEEDED. Cmd+U: `PromptBuilderTests` 13/13 зелёных (вкл. 4 новых), `ProductBlockSeederTests` 2/2 зелёных — пользователь подтвердил выводом. Запуск приложения (UI-тесты, многократный launch) — без краша миграции схемы. Сквозной поток «выбор блока → запрос к ИИ» руками отдельно не гонялся (покрыт юнит-тестами `PromptBuilder`).
