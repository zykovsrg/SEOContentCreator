### 2026-06-21 — Под-проект «Публикация в Google Docs» (реализация на ветке)

- Change: Реализован под-проект «Публикация» на ветке `feature/publishing-google-docs` (9 задач, TDD, subagent-driven). Новая SwiftData-модель `ExternalDocument` (история публикаций) + связь `Topic.publications`. Логика (всё на URLSession, без сторонних зависимостей): `GoogleCredentialStore`/`GoogleTokens` (ключи и токены в Keychain), `MarkdownDocParser` (markdown→блоки), `DocsRequestBuilder` (блоки→запросы `batchUpdate`, индексы UTF-16), `GoogleDocsClient` (REST Docs/Drive с retry/backoff), `GoogleAuthService` (свой OAuth desktop + PKCE + loopback на `127.0.0.1`, скоупы documents + drive.file), `ArticlePublisher` (оркестратор: новый документ / перезапись). UI: раздел «Google Docs» в Настройках (Client ID/Secret, вход/выход), кнопка «Опубликовать» в тулбаре `TopicWorkspaceView` + `PublishSheet` (превью, выбор режима, история, перезапись с подтверждением). Публикация реализована как действие/кнопка, НЕ как case в `PipelineStage`. Документы кладутся в папку Drive «SEO-статьи клиники».
- Impact: Закрыт финальный этап пайплайна «от идеи до выпуска». Компиляция тестовой сборки зелёная (`build-for-testing`). Реальный прогон тестов (Cmd+U) и реальная публикация ещё не выполнены — нужна настройка OAuth-клиента в Google Cloud. Ветка ещё не влита в `main`.
- Manual checks: PENDING (требуют Google Cloud Desktop OAuth-клиента): вход в Google; публикация (новый/перезапись/новый ещё раз); попадание в папку; формат документа (заголовки/списки/жирный); поведение при ошибке сети. Известный пункт на упрочнение: `LoopbackListener` слушает `NWListener(on: .any)` — стоит ограничить только loopback-интерфейсом.

### 2026-06-20 — под-проект 8 «Генерация изображений»

- Change: Реализован под-проект 8 (17 задач по плану, TDD). Новые SwiftData-модели: `GeneratedImage` (`@Attribute(.externalStorage)` для данных), `ImageStylePreset`, `ImagePromptTemplate`. Расширена модель `Topic` (поля `images`, `coverImageID`). Логика: `ImageClient` (generations + edits endpoints, multipart для edits), `ImageResponseParser`, `ImageGenerator` (@Observable, рендер в превью + лог GenerationJob), `ImageSaver` (persist в галерею), `ImagePromptBuilder` (сюжет + стиль), `ImagePromptDefaults`, `ImageStylePresetDefaults` (брендовая палитра). Засев шаблонов промтов и пресета по умолчанию в `StageTemplateSeeder`. Переменная `{{выделенный_фрагмент}}` в TemplateVariables. UI: `ImageGenerationSheet` (создание обложки / иллюстрации / доработка), `ImagesView` (галерея + экспорт PNG + архив), кнопка «Изображения» в тулбаре `TopicWorkspaceView`, раздел «Изображения» в `TemplatesView` (CRUD промтов и пресетов), поле модели изображений в `SettingsView`. Имя модели настраивается через `@AppStorage("imageModel")`; провайдер — OpenAI images API. Дополнительно: багфикс гонки тест-изоляции в `ImageClientTests` (собственный `MockImageURLProtocol`).
- Impact: Тема теперь имеет галерею изображений (обложка + иллюстрации), отдельную от ленты текстовых версий. Иллюстрация сохраняет якорь-цитату для будущей вставки в Google Docs. Доработка только промтом (без маски); новая картинка не затирает исходную — `sourceImageID`. Коммит с фиксом изоляции тестов: `6fc2543`. Ветка: `codex-subproject-8-images`.
- Manual checks: `xcodebuild build-for-testing` — TEST BUILD SUCCEEDED. Cmd+U: 123/123 тестов зелёных (35 suite). Smoke (Cmd+R, реальный ключ) — пользователь подтвердил: обложка, иллюстрация, доработка, экспорт PNG, переключение обложки, архив, редактирование пресетов/промтов, ошибка при неверном ключе — всё работает.

### 2026-05-24 — под-проект 7 «Структура» (этап плана отдельно от текста)

- Change: Добавлен первый этап пайплайна «Структура» (роль `author`): ИИ-автор генерирует план H1/H2/H3 с краткими пометками по разделам, без ключевых запросов. План хранится в редактируемом поле темы `Topic.structureText` (по образцу «Семантики») и правится в новом окне `StructureEditorSheet` (кнопка «Сгенерировать» + ручная правка). Новая переменная `{{структура}}`; этап «Черновик» переписан — пишет текст по плану (мягкий фолбэк: пустой план → пишет с нуля). Генерация структуры не создаёт версию в ленте (`StageExecutor` ветка для `.structure`). Новый этап досевается автоматически; миграция дефолтов не нужна.
- Impact: Логика ИИ-автора стала последовательной: редполитика → источники (через контекст роли) → структура → текст. Появилась контрольная точка на плане до генерации полного текста. Изменена модель `Topic` (новое поле) — лёгкая миграция SwiftData. Семантика осталась отдельным поздним этапом. Следующий — под-проект 8 «Изображения» (обложка + иллюстрации + стиль/пресеты) под Superpowers.
- Manual checks: `xcodebuild build-for-testing` — TEST BUILD SUCCEEDED (приложение + unit + UI). Добавлены тесты: `{{структура}}` в PromptBuilder, структура без версии в StageExecutor, «Черновик» использует `{{структура}}` и «Структура» без ключей в StageTemplateDefaults, «Структура» первый/author в PipelineStage. Cmd+U и smoke (Cmd+R) — за пользователем.
