## Goal

Сбор семантики (`SemanticCollectionRunner.run`) не должен терять весь прогресс
при обрыве соединения или ошибке Yandex Wordstat. Нужны две связанные вещи:

1. Сохранение прогресса между этапами сбора (какие seed-фразы уже опрошены,
   что уже получено), чтобы после обрыва/перезапуска можно было продолжить с
   последнего пройденного шага, а не с начала — плюс явная возможность
   пользователя сбросить сохранённый прогресс и начать заново.
2. Если Wordstat/Yandex отвечает ошибкой или соединение рвётся, весь прогон
   должен останавливаться целиком (а не тихо продолжать перебирать
   оставшиеся seed-фразы, как сейчас), чтобы не жечь время и лимиты впустую
   на заведомо недоступном API.

## Done criteria

- Если Wordstat возвращает ошибку API или соединение обрывается, текущий
  прогон сбора семантики останавливается целиком (не продолжает перебирать
  оставшиеся seed-фразы) и сообщает пользователю понятную причину остановки.
- Прогресс уже опрошенных seed-фраз/полученных Wordstat-результатов
  сохраняется так, что повторный запуск сбора для той же темы продолжает с
  последнего пройденного шага, а не запрашивает всё заново.
- У пользователя есть явный способ сбросить сохранённый прогресс для темы и
  начать сбор с нуля.
- Существующее поведение не ломается: funnel-журнал, чанкованное сохранение
  (`recordInChunks`), лимит времени 10 минут, кнопка «Остановить», откат при
  ошибке сохранения — всё продолжает работать как раньше.
- Юнит-тесты покрывают: остановку на ошибке Wordstat, возобновление с
  сохранённого шага, явный сброс прогресса.
